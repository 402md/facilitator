# Git Workflow — Planning Workspace

## Commit Messages

- Format: `<type>: <description>` (lowercase, imperative, no period)
- Types: `docs`, `chore`, `fix`
- Describe the "why", not the "what"

Examples:

```
docs: add competitive analysis for tempo settlement
docs: update api design with batch endpoint
fix: correct cctp fee calculation in spec
chore: add implementation plan phases 3-4
```

## No Co-Authored-By

Never add `Co-Authored-By` lines to commits.
